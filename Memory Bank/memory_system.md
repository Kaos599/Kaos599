# Memory System - Kaos599 README

## Visual Memory Concept
While the README is technically "static", the visual memory is simulated through:
1.  **Persistence of Style**: Consistent usage of Chartreuse and specific line weights across all components.
2.  **Progressive Typing**: The "About" section simulates a memory retrieval process through typing animations.
3.  **Stateful Hovers**: Using `:hover` to reveal "hidden" system information (Easter Eggs).

## Technical Note
Since GitHub strips all persistent client-side state (JS, Cookies, LocalStorage), all "memory" is hard-coded into the SVG animations or the Markdown structure itself.
